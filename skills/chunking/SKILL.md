# Chunking

## Overview

A massive, vague task is hard to start because there is no obvious first move. The fix is not a better grand plan. A forty-item plan is itself overwhelming, and it is also task-reordering bait. The fix is one concrete next action small enough that starting it is easy.

## When this triggers

Fires when your human partner is stuck at the front of a task: circling it without starting, calling something overwhelming or huge, or holding a big goal with no visible entry point.

## How to chunk

1. **The deliverable is the next action, not the plan.** The question to answer is "what can you start in the next ten minutes", not "what are all the steps".
2. **Make it genuinely startable.** Concrete means unambiguous about the very first move. "Set up the repo" is still vague. "Create the repo and add a README stub" is startable. If the proposed action still feels big, chunk it again.
3. **One next action usually beats five.** Five next steps is a small plan, and a plan is the thing that was overwhelming. Give one. The next one can come once there is momentum.
4. **Do not produce the full plan unless asked.** If they want the whole map, they will ask. Defaulting to it reintroduces the overwhelm and invites reordering.
5. **Reduce the activation cost.** The hardest part is the transition into the task. Strip scaffolding, preamble, and setup that can wait. See `transitions` for the start-of-task case.
6. **Name done for the chunk.** A small action with a clear finish line is easier to start than an open-ended one. Say what completing this chunk looks like.

## Red flags

| Symptom | What it means |
|---|---|
| You produced a grand plan when asked for a starting point | Wrong deliverable. Give the next action. |
| The "next action" cannot be started in a few minutes | Not chunked enough. Chunk again. |
| You gave ten next steps | That is a plan, and a plan was the problem. Give one. |
| You explained the whole project | They needed an entry point, not a tour. |
| The action has no clear finish | Open-ended chunks are hard to start. Define done. |

## Rationalizations to reject

- "They need to see the whole plan to feel oriented." They need to start. Momentum orients better than a plan.
- "This action is concrete enough." If they cannot begin it in a few minutes, it is not. Chunk again.
- "One step is too small to bother with." One started step beats ten planned ones.
