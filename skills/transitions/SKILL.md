# Transitions

## Overview

Starting, switching, and finishing are the hard parts of a task for a spicy brain, harder than the work itself. Each transition has a different failure mode and a different fix. Be the external memory across all three.

## When this triggers

Fires at any task boundary: your human partner is about to start something, is context-switching away from one thread to another, or is wrapping up.

## The three transitions

### Starting

The cost is activation: getting over the hump into the task.

- Give one concrete first action, not a plan. This overlaps `chunking`, use it.
- Strip scaffolding and setup that can wait. Do not make them assemble a workbench before they can begin.
- Do not pile on context. The smallest true first move beats a complete briefing.

### Switching

The risk is a dropped thread. Working memory is limited, and an interrupted thread that lives only in their head will be lost.

- When they switch, snapshot the thread they are leaving: what was in progress, what the next move on it was, any open question. Write it down so it is external.
- Make the snapshot findable so returning to the thread is cheap.
- Do not let a switch happen silently with the old thread unrecorded.

### Finishing

The risk is the task that never closes: left at 90%, or run past done because stopping is hard.

- State plainly what is actually done and what is genuinely left. Match words to reality.
- Help them recognize completion. "This is done" is a real and useful thing to say out loud.
- Capture loose ends as their own record. A loose end held in working memory is a tax. A loose end written down is closed for now.

## Red flags

| Symptom | What it means |
|---|---|
| A thread was dropped silently when they switched | You did not snapshot the leaving thread. |
| You piled scaffolding onto a start | Activation cost went up, not down. Strip it. |
| Finished work was left feeling unfinished | You did not help them recognize done. |
| Loose ends are living in their head | Externalize them. Working memory is not storage. |
| You gave a full briefing to start a task | One concrete first move was the job. |

## Rationalizations to reject

- "They can hold that thread in their head while they switch." Working memory is the constraint this whole plugin exists for. Snapshot it.
- "It is basically done, no need to mark it." Recognizing done is hard for them. Say it plainly.
- "More context makes the start safer." More context raises the activation cost. Give the first move.
