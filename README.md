# spicypowers

A Claude plugin of skills for working with a *spicy brain* - a neurodivergent, parallel-processing, hyperfocus-driven way of thinking. It is the operating layer: how to communicate, intake unstructured input, translate work into status, reduce task-transition friction, pick the right kind of thread, and build trackers that hold non-linear state.

None of these skills require writing code, which is the line between this plugin and its sibling `nominalpowers` (the engineering layer). spicypowers is usable by anyone who works with a Claude, technical or not.

## Skills

`spicypowers` ships ten skills. One is always-on, nine are situational.

**The spine (keep active for the whole session):**

- **using-spicypowers** - establishes how a spicy brain works, the always-on communication style, the shared vocabulary, and the roster below. Read it first.

**Situational (each triggers on its own):**

- **brain-dump-intake** - reassembling unstructured, parallel, out-of-order input into signal without dropping threads or reordering priorities.
- **sbtb** - the Spicy Brain Translation Bureau. Brain dump in, manager-safe status update out. Also a non-linear capture layer.
- **chunking** - turning a massive vague task into the next concrete, startable action.
- **transitions** - reducing friction at the three hard task boundaries: starting, switching, finishing.
- **nominal-voice** - mechanical writing conventions for anything sent under your own name, so it reads as human-written.
- **tracking-artifacts** - building persistent, cross-device interactive trackers with categorized ticket IDs.
- **research-adventures** - running and logging a structured technology or approach evaluation.
- **riddle-me-this** - handling an exploratory feasibility probe without mistaking it for a build order.
- **thread-archetypes** - choosing the right kind of thread for the context, as a focus tool, not a hard boundary.

## A note on names

These skills are written generically ("your human partner") so the plugin is shareable. The Claude installing it already knows who it is talking to, so no name configuration is needed.

## Installation

Built for the Claude plugin/skills system. Install the packaged plugin through your client's plugin or skills settings. Skill folder names follow kebab-case; each skill is a directory under `skills/` containing a `SKILL.md`.
